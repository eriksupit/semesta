## The Granular Prompt Standard (mandatory all scenes)

**Scene WITH human character (S1–S17):** two FIXED blocks + VARIABLE fields.
- `character_identity` (FIXED, copy-paste verbatim across same character's scenes): role, ethnicity, beauty, age, facial_features, body_features
- `character_style` (FIXED): makeup, wardrobe, hair
- VARIABLE: scene, subject_state, action, framing, pose, gesture, expression, angle, camera, lighting, mood, style, color_grade, aspect_ratio

**Scene WITHOUT human (S0 / inanimate):** same granular structure, adapted.
- `subject_anchor` (FIXED): primary_subject, subject_material, supporting_objects, human_absence_signal
- `pose` → `composition`; `gesture` / `expression` omitted
- All cinematography fields identical to character scenes

**Field mapping (character → inanimate):** character_identity→subject_anchor; role→primary_subject; facial_features→subject_material; body_features→supporting_objects; (new) human_absence_signal; pose→composition.

## Critical Rules Distilled (full detail in prompt-rules file)

- Token strings, NOT prose. C2 English, adjective-heavy, zero filler/copula.
- NEVER "Indonesian" anywhere — use city/ethnicity (Jakarta, Javanese, Sundanese, Batak, Makassarese).
- Role environment-agnostic — no "TV presenter"/"doctor"/"commoner".
- Beauty token in Bahasa Indonesia, pick ONE: cantik/jelita (F), tampan/gagah (M).
- Age = single explicit number, +5yr aging-bias compensation (target 35 → "20 years old").
- Skin tone: "warm ivory-olive, luminous" for polished — never "brown".
- Makeup = 3 layers: base + features + FINISH (matte, zero shine). Finish layer prevents oily skin. BANNED "HD makeup" (specular shine).
- Wardrobe = explicit fashion term, cool-neutral (slate blue/charcoal) for tone-neutral scenes — warm/terracotta pulls grade yellow.
- Hair = styling terminology with "glossy finish".
- framing: shot-size terms; live actors need "full frame knees to head, floor visible bottom edge" to prevent waist-up drift.
- pose: portrait terms (contrapposto default); gesture: explicit hand position; expression: PHYSICAL description not emotion label.
- subject_state mandatory first: static/dynamic/speaking.
- mood mandatory directorial instruction per beat.
- angle separate from camera; "Panavision Ultra Primo anamorphic" mandatory all prompts.
- COLOR GRADE = ONE SYSTEM all scenes: Roger Deakins palette, lifted blacks medium gray, cyan-teal shadows, clean whites, Kodak 2383 — CONSTANT. Only midtones temperature shifts. BANNED "amber-ochre shadows".
- Photographer/DP anchors: Gregory Crewdson (empty), Annie Leibovitz (character portraits), Steve McCurry / Bradford Young (documentary local), Emmanuel Lubezki (outdoor S4/S6/S12), Hoyte van Hoytema (corporate S8/S14), Roger Deakins (color grade all).
- Action = mid-process state. Empty scenes add human_absence_signal (steam mug, pen diagonal).

## Generation Workflow (every scene)

1. Claude converts the scene's prose DRAFT (or reference note) into a full granular JSON prompt following the standard.
2. Claude presents the prompt. User generates in ChatGPT (GPT Image 2).
3. User sends the generated image back.
4. Claude reviews: APPROVE → log to `08` + rename file + next scene. NOT approved → diagnose the specific rule violation (tone too yellow = color_grade; oily skin = makeup finish layer; waist-up = framing), fix ONLY the wrong field, re-present, repeat.
5. After approve: user drags the file into `explainer-video/rough-presentation-image/`, Claude renames via Filesystem:move_file to `S{n}-{slug}.png`.
6. Log every approval in `08-investor-deck-brief.md` revision log. Add every new lesson to `prompt-rules-image-generation.md`.

Claude does NOT generate images. Claude writes prompts and reviews results.

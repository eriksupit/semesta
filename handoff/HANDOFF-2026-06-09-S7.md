## 3. STANDAR PROMPT BAKU (dari S2 — WAJIB DIIKUTI)

Sumber kebenaran tunggal: **`prompt-rules-image-generation.md`** (root vault). Baca penuh sebelum generate apapun. Ringkasan pola baku:

### Urutan field (top-to-bottom)
```
mood → color_grade → style → character_identity → character_style → scene →
subject_state → action → [on_screen_graphic + graphic_proximity] →
framing → pose → gesture → expression → angle → camera → lighting → aspect_ratio
```

### Aturan kritis yang sudah divalidasi
1. **Token warna HANYA di `color_grade` dan `mood`.** Zero warna di `facial_features`, `wardrobe` (pakai colorway desaturated), `scene`, `lighting`.
2. **`facial_features`** = kualitas/tekstur saja (`healthy glow`, `luminous complexion`, `clear healthy skin`), zero nama warna.
3. **`wardrobe`** = garment construction tokens (fabric + construction + fit + colorway desaturated). BANNED: batik/paisley/floral/ethnic print, terracotta/ochre/amber/golden.
4. **`scene`** = design interior tokens (material finish + fixture + surface). BANNED: label geografis/etnis (`Jakarta kitchen`, `traditional Indonesian`).
5. **Color grade satu sistem:** Roger Deakins palette. Hanya midtones bergeser: `cool midtones desaturated` | `slightly warmer midtones` | `neutral cool midtones`.
6. **Negasi DILARANG** di semua field (`no X`, `not X`, `un-`, `non-`). Gunakan token positif. Contoh: bukan "not on phone screen" tapi "air-composited superimpose over scene".
7. **`age`** = satu angka eksplisit, +5 tahun aging bias (target 35 → tulis "20 years old").
8. **`ethnicity`** = etnis spesifik (`modern Javanese features`) atau urban identity (`modern Jakartan features`). BANNED: `Indonesian features`, `Betawi features`.
9. **`beauty`** dikalibrasi kelas sosial: `jelita`/`tampan`/`gagah` (middle ke atas), `cantik` (lower-middle/domestic).

### Teknik UI floating graphic (Teknik 2) — POLA BAKU S2
Untuk scene dengan graphic UI mengambang (S4 Teknik 2; S6/S10/S12/S14 Teknik 3 embedded; S8 Teknik 1 ambient glow saja):

**Dua field berpasangan WAJIB:**

`on_screen_graphic` = konten + visual treatment:
```
"air-composited superimpose over scene — frosted glass card panel rounded corners semi-transparent white fill subtle inner glow edge soft gaussian blur backdrop — element 1: '[teks utama]' clean sans-serif large weight teal-cyan accent color centered on card — element 2: '[badge]' pill-shaped badge teal-cyan solid fill white text medium weight centered below element 1 — card panel soft drop shadow layered depth over scene."
```

`graphic_proximity` = **CSS positioning logic** (BUKAN arah manusia — model tidak baca atas/bawah/kiri/kanan):
```
"center-x aligned with [device], top edge of card 20px above top edge of [device screen], card width approximately [N]px, card height approximately [N]px, z-index above subject hands, z-index below subject face."
```

**Pola S2 yang tervalidasi (template untuk Teknik 2):**
```
graphic_proximity: "center-x aligned with smartphone device, top edge of card 20px above top edge of smartphone screen, card width approximately 280px, card height approximately 100px, z-index above subject hands, z-index below subject face."
```

**BANNED untuk UI:** `ui_overlay`/`floating_ui`/`screen_ui` (trigger UI di dalam layar gadget), `phone screen facing camera` (trigger gadget dibalik demonstrasi), pixel offset >80px (disconnected), arah manusia tanpa anchor+pixel.

**Catatan:** S6/S10/S12/S14 di `08` masih pakai `ui_overlay` (Teknik 3 embedded di device — sah untuk scene itu karena UI memang di layar device, bukan mengambang). S4 sudah pakai `on_screen_graphic` + `graphic_proximity` Teknik 2. Verifikasi per scene saat generate.

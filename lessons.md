## Preferensi & Gaya Kerja (Tervalidasi)

- User menolak menu / multiple options tanpa rekomendasi tunggal. Selalu tutup dengan SATU rekomendasi + satu approval gate.
- User tidak suka kata "bayangkan" — hindari dalam semua copy.
- User menuntut riset sebelum rekomendasi apapun — jangan jawab dari memori tanpa verifikasi.
- Kata "terserah kamu" dari user = sinyal bahwa Claude gagal riset atau gagal berpikir mandiri. Lakukan riset dulu, jangan terima.
- User sering menjawab singkat: "approve", "kunci", "go", "lanjut", "oke" = persetujuan penuh. Langsung eksekusi.
- User tahu tools-nya — tidak perlu penjelasan panjang, langsung ke keputusan.
- Setiap keputusan harus masuk ke file vault — tidak boleh mengambang di chat history.
- Frontmatter Obsidian wajib lengkap di setiap file yang dibuat.
- Project instructions Claude harus full English.
- Pipeline memory wajib dijalankan di akhir setiap sesi.
- User generate gambar eksternal (ChatGPT/GPT Image 2), Claude menulis prompt + review + approve. Claude tidak generate gambar sendiri.
- User sangat presisi soal tone/mood visual — diagnosa harus identifikasi rule violation spesifik, bukan tebakan umum.
- Ketika user marah dan menunjuk kesalahan spesifik — jangan defensif, langsung akui, fix, catat ke vault. Koreksi user adalah lesson paling berharga.
- **Pembagian peran: SAYA penulis konten, Claude Design HANYA pelayout.** Jangan menyerahkan kerja konten (menarik/mengonfirmasi angka dari brief mentah, menyusun isi chart, menulis copy) ke agen desain — itu domain saya. Pola benar: saya buat SATU dokumen konten FINAL design-ready (tiap slide: headline+body+visual-spec+speaker, semua angka sudah eksplisit di dalamnya, verbatim), lalu handoff/initial-prompt menyuruh Claude Design "design-only, layout verbatim, jangan menulis/menghitung, jangan upload brief mentah". Jangan upload Executive Brief/Proyeksi NU ke agen desain — ekstrak semua yang perlu ke doc konten. Validated: `11-adprovider-deck-content.md`.
- **Cek AUDIENS deck ke `01-concept-document §2` SEBELUM menulis copy/handoff.** Explainer video + deck turunannya = audiens **PROVIDER IKLAN** (Google/Meta/agensi/jaringan iklan), rantai nilai: video→provider iklan lihat potensi traffic & inventory→approach brand→brand iklan→biayai pre-development. BUKAN investor penggalang dana. Saya sempat membangun deck naratif `09` sebagai investor capital-raise (capital ask, Series A/B, equity) — salah audiens. Provider iklan peduli: skala audiens, engagement, jangkauan distribusi (NU), brand-safe environment, belonging ("brands that mean something, not just shown"). Register traffic/inventory/reach BOLEH untuk deck provider iklan (kebalikan dari larangan di deck investor `08`). ## Insight Prompt Image Generation (Universal — Semua Model)

*File lengkap: `prompt-rules-image-generation.md` di root vault. Wajib dibaca sebelum menulis prompt apapun.*

- Jangan skip bootstrap di awal sesi.
- Jangan copy-paste cinematic video prompts ke illustration prompts.
- Jangan lock illustration prompts sebelum user approve.
- Jangan skip pipeline memory update di akhir sesi.
- Jangan jalankan /session-anchors tanpa empat companion skills.
- Jangan gunakan "editorial illustration" sebagai style keyword.
- Jangan gunakan "cinematic digital painting" — menghasilkan dark surveillance room.
- **Token UNIFORM, bukan gradien — di set konsistensi-kritis (plat referensi).** Gradien/gradasi (`graying temples`, fade, sheen-falloff, peralihan warna/tekstur) di-render beda tiap generate → titik inkonsistensi. Tulis state uniform tunggal & POSITIF: `uniform/even/consistent/flat <X> throughout` (mis. `salt-and-pepper graying temples` → `uniformly salt-and-pepper hair, even grey distribution throughout, flat matte finish`). Larangan jangan ditulis `no gradient` (negasi) — pakai token uniform positif yang setara makna. Kunci terkuat: salin via reference-image. (Koreksi user 2026-06-19 atas sheen rambut close-up Herman; dipertegas di `prompt-rules` + memory.)
- **Keyword-leak — bukan hanya `no`/`non`, hindari kata-dominan yang menyeret konsep tak diinginkan.** GPT Image 2 autoregressif (generate progresif, tak bisa mundur koreksi) → konsep salient terbentuk duluan. `light-absorbing` bocorkan `light` (render cahaya), `low-sheen` bocorkan `sheen`, `non-reflective` bocorkan `reflective`. Pakai token akar-bersih. **Token matte-rambut-gelap terkunci:** `dry matte hair, velvety matte strands, chalky matte finish, loosely combed natural`; lighting plat buang `high-key` (pemicu specular) → `flat even soft diffused three-point studio key … matte skin and hair rendering`. Uji tiap token: bila satu kata diambil sendiri memunculkan hal tak diinginkan, ganti. (Koreksi user 2026-06-19, sheen rambut Herman.)
- **Jangan gunakan prompt negatif — MUTLAK & TEGAS.** `no X`, `not X`, `not in frame`, `without X`, `un-`, `non-`. Model membaca negasi sebagai "JANGAN itu, lakukan ini" → mengambil inisiatif sendiri → non-compliant. **Tulis POSITIF (apa yang ADA), atau HAPUS field bila elemen tak tampak karena framing.** Pola: `tidak berkepala`→`headless`; `no stylized push`→`faithful natural rendering`; sepatu di close-up → hapus field `footwear`, jangan `"not in frame"`. (Koreksi user 2026-06-19 atas plat Herman; dipertegas di `prompt-rules` core principle + BANNED MASTER LIST + `08-character-sheet/00-README §3`.)
- **Jangan tulis prompt sebagai prose panjang dengan kalimat utuh** — kata sambung dan copula diabaikan model. Gunakan token strings padat.
- **Jangan tulis prompt tanpa geographic anchor** — model default ke Western aesthetic tanpa Jakarta/tropical tokens.
- Jangan rekomendasikan format prompt tanpa riset.
- Jangan terima "terserah kamu" — lakukan riset dulu.
- Jangan gunakan framing ad-provider dalam investor copy.
- Jangan isi Slide 17/18/20 tanpa label ⚠️ PLACEHOLDER.
- Jangan rekomendasikan unofficial Midjourney API.
- Jangan asumsikan Gemma 4 bisa generate gambar.
- Jangan gunakan "Sicario register" tanpa nama DP — terlalu loose.
- **Jangan sebut "Indonesian" di field karakter manapun** — 1.300+ etnis → model bingung, hasil generik. Pakai kota/etnis spesifik (Javanese, Sundanese, Batak, Makassarese) atau urban city identity (Jakartan).
- **Jangan pakai role yang trigger environment** — "TV presenter"/"news anchor"/"doctor" memaksa studio/klinik, override scene. Pakai "Jakarta professional woman".
- **Jangan pakai rentang usia** — inconsistent. Satu angka eksplisit, kompensasi aging bias +5 tahun (target 35 → tulis "20 years old").
- **Jangan pakai token warna di `facial_features`** — `character_identity` dibaca model lebih dulu dari semua field lain. Token warna di sini (golden, ivory, tan, warm, brown, beige) override seluruh palette sebelum `color_grade` sempat dibaca. `facial_features` hanya berisi tekstur dan kualitas: `luminous complexion`, `healthy glow`, `radiant complexion`, `clear healthy skin`. Warna dikontrol sepenuhnya oleh `color_grade` dan `mood`.
- **Jangan tulis makeup tanpa finish layer** — skin oily/lusuh. Tiga layer wajib: base + features + finish (baked setting powder, matte finish, zero shine).
- **Jangan pakai "HD makeup"/"HD setting powder"** — trigger high-contrast specular highlights, kulit berkilau di titik cahaya.
- **Jangan pakai wardrobe warm/terracotta untuk scene tone-neutral** — pull color grade ke kuning. Pakai slate blue/charcoal cool-neutral.
- **Jangan pakai token warna eksplisit di `color_grade`** — `golden`, `amber`, `ochre`, `warm` sebagai standalone color descriptor override Roger Deakins sistem. Token midtones valid hanya: `cool midtones desaturated`, `slightly warmer midtones`, `neutral cool midtones`.
- **Jangan biarkan framing drift ke waist-up** untuk live actor speaking — tambah eksplisit "full frame knees to head, floor visible bottom edge".
- **Ethnicity bukan harus etnis spesifik** — untuk karakter urban lintas-etnis, pakai urban city identity: `"modern Jakartan features"`. Ini berbeda dari etnis spesifik (Javanese, Sundanese). BANNED: `"Betawi features"` sebagai representasi umum warga Jakarta.
- **Kelas sosial masuk ke `role`** — format: `[kota] [kelas sosial] [deskripsi peran]`. Kelas sosial mengkalibrasi grooming level, appearance register, dan beauty token. `jelita` = middle-class ke atas. `cantik` = lower-middle class, domestic.
- **Wardrobe = garment construction tokens wajib** — label motif (`batik`, `paisley`, `ikat`, `floral`) trigger stereotip lokal. Wajib: fabric specification + garment construction detail + fit/silhouette + colorway desaturated.
- **Scene = design interior tokens wajib** — label geografis/etnis (`Jakarta kitchen`, `traditional Indonesian interior`, `Javanese decor`) trigger stereotip. Wajib: material finish + fixture type + surface specification.
- **Urutan field: `mood → color_grade → style` di atas `character_identity`** — model baca JSON top-to-bottom. Tone/palette harus terkunci sebagai anchor global sebelum karakter dibaca.
- **`on_screen_graphic` untuk Teknik 2 floating graphic** — field resmi untuk elemen tipografi mengambang di udara (frosted glass card, teal-cyan accent, pill badge). Selalu berpasangan dengan `graphic_proximity`.
- **`graphic_proximity` = CSS positioning logic, bukan arah manusia** — model image generation membaca koordinat visual seperti CSS/compositing engine. Wajib: anchor element + pixel offset + pixel dimensions + z-index. Pattern baku tervalidasi S2: `center-x aligned with smartphone device, top edge of card 20px above top edge of smartphone screen, card width approximately 280px, card height approximately 100px, z-index above subject hands, z-index below subject face.` BANNED: arah relatif manusia tanpa anchor+pixel, pixel offset >80px.

## Arsitektur Prompt Granular (Standar Wajib Semua Scene — Sesi 5)

- Struktur granular S1 = standar wajib SEMUA 18 scene. Konsistensi struktur = konsistensi visual.
- Scene karakter: dua blok FIXED (`character_identity` + `character_style`) + field VARIABLE.
- Scene benda mati: `subject_anchor` (primary_subject, subject_material, supporting_objects, human_absence_signal) menggantikan blok karakter; `pose`→`composition`; `gesture`/`expression` dilewati; field sinematografi identik.
- `character_identity`/`character_style` setelah approved = dikunci, copy-paste verbatim ke scene karakter sama.
- Field VARIABLE per scene: scene, subject_state, action, framing, pose, gesture, expression, angle, camera, lighting, mood, color_grade.
- `subject_state` (static/dynamic/speaking) = token pertama wajib, mengunci kondisi dasar sebelum action.
- `expression` = deskripsi fisik ("mouth open mid-speech, lifted brows animated"), bukan label emosi ("happy").
- `mood` = instruksi direktorial per narrative beat, wajib.
- `angle` terpisah dari `camera` (angle = posisi kamera, camera = lens/DOF).
- Setiap field punya token library tervalidasi di `prompt-rules-image-generation.md`.

## Insight RAB & Budgeting Produksi (Sesi 9 Juni 2026)

- **Honor tenaga ahli = model tiga-lane terdiferensiasi, bukan rata.** Bayar sesuai bobot beban kerja per lane, bukan flat. Director jangkar Rp 25 jt (lintas semua fase + naskah); eksekutor dikalibrasi 45–56% director. Dikunci: Director 25 / Editor-Picture-lock 14 / AI Generation Artist 13 / Motion Graphics & Finishing 11 jt = Rp 63 jt. Dasar: editor pikul integrasi timeline terberat + engagement terpanjang; AI gen volume iterasi tertinggi + premium kelangkaan skill; motion graphics spesialis tapi window pendek.
- **RAB tools (`03`) dan honor manusia dan syuting fisik (`03-a`) wajib dipisah tegas — jangan dobel hitung.** Director memimpin syuting → honornya di `03`, BUKAN di crew `03-a`. Crew lapangan `03-a` murni freelancer hari-H.
- **Live actor = set tetap berdandan, BUKAN greenscreen+AI background.** Tiga alasan terkunci: (1) brief minta environment hangat bertekstur + rack focus, mustahil di greenscreen datar; (2) live actor adalah jangkar manusiawi yang dikontraskan dengan dunia AI — compositing manusia ke background AI meruntuhkan kontras inti video; (3) compositing manusia ke atas AI *video* (bukan still) = VFX spesialis berisiko tinggi (flicker temporal, lighting mismatch cool-key vs warm-bg), membengkakkan post. Greenscreen menang HANYA kalau butuh banyak latar berbeda — di sini cuma butuh 1 environment konsisten, set tetap lebih murah+andal.

## Insight Konversi Dokumen ke DOCX (Sesi 9 Juni 2026)

- **Strip frontmatter aman = hanya blok awal, jangan hitung semua `^---$`.** Dokumen vault memakai `---` sebagai pemisah seksi (thematic break) secara masif. Pola "hitung dua `---` lalu print sisanya" akan salah potong body. Pola benar (awk): set flag hanya jika `NR==1 && /^---$/`, tutup di `---` berikutnya, print sesudahnya — `---` selanjutnya aman.
- **Verifikasi tabel docx via unzip, bukan roundtrip HTML.** `pandoc docx -t html | grep '<table>'` bisa keliru (atribut tag). Cek otoritatif: `unzip -p file.docx word/document.xml | grep -o '<w:tbl>' | wc -l`. Pandoc me-render pipe table markdown jadi tabel Word asli — terkonfirmasi.
- **Humanisasi Level 2 untuk dokumen data (RAB/brief) = glosarium-di-depan, BUKAN tulis-ulang prosa.** Menulis-ulang tabel angka berisiko mengubah substansi (langgar scope). Pola aman: salin body verbatim + sisipkan bagian "Panduan Membaca & Glosarium Istilah" di depan. Memenuhi syarat "jargon dijelaskan + terbaca awam" dengan integritas data nol-perubahan.
- **Bahasa dokumen tidak seragam.** 01/02/03/03-a ditulis Bahasa Indonesia (prosa ID + jargon EN menempel); 05/06/07 + Section 1–4 dari 08 berbahasa Inggris. Aturan CLAUDE.md "dokumen dalam English" tidak ditegakkan seragam di output lama. Untuk humanisasi: ikuti bahasa asli tiap dokumen, jangan terjemahkan (terjemahan = scope creep + risiko ubah makna).
- **Prompt JSON image-generation bukan konten awam.** Section 5 dari `08` (~960 baris token-string) tidak bisa di-"prosa-kan"; perlakukan sebagai lampiran teknis berlabel dan jelaskan cara membacanya di glosarium, bukan dihumanisasi.

## Insight Kling AI — Model & Kredit (Sesi 2026-06-17, terverifikasi web)

*Detail lengkap + cross-check sumber: memory `semesta-kling-pricing-models.md`. Anchors dibuat eksternal di ChatGPT Image v2, video di Kling via teknik start→end keyframe.*

- **Paket Premier = kantong kredit, BUKAN pengunci model.** $64.99 / 8.000 kredit. Bisa jalankan 2.6 maupun 3.0; bayar per-generate. Kling 3.0 sudah GA (rilis 5 Feb 2026), tidak lagi eksklusif Ultra.
- **Konsistensi karakter = fitur andalan 3.0 yang 2.6 TIDAK punya sama sekali.** 3.0 = penguncian via 3–5 gambar referensi (Elements/Director Mode) → karakter tidak morphing antar-scene. 2.6 hanya andalkan prompt + seed (tidak presisi). Ini menyerang akar masalah success-rate 50% wajah manusia di RAB.
- **Tarif kredit Kling 3.0 image-to-video (per detik):** 720p tanpa audio = 6; 1080p tanpa audio = 8; **1080p DENGAN audio = 12** (5 dtk=60, 10 dtk=120). Audio WAJIB per keputusan user → pakai 12 kredit/detik.
- **Kapasitas 1 subscribe Premier (8.000 kredit, 1080p audio-on):** KOTOR 666 dtk (~11 mnt). BERSIH: 50%→333 dtk, 65%→**433 dtk (~7 mnt)**, 70%→466 dtk. Kebutuhan riil Kling hanya ~90 dtk (3 scene dinamis) → satu subscribe ~3,7× berlebih; "subscribe 2×" = cadangan revisi, bukan kekurangan kapasitas.
- **Segmen 5 detik = STANDAR PRODUKSI.** Per-kredit identik dengan 10 dtk (linear, tanpa ongkos-start), TAPI generate gagal hanya buang 60 kredit (vs 120) → memangkas pemborosan per kegagalan separuh + kontrol start→end keyframe lebih ketat. Trade-off: lebih banyak sambungan editing (ditekan oleh teknik keyframe).
- **Angka success-rate 50%/65% adalah benchmark generasi LAMA** — wajib di-benchmark ulang pada 10–15 generate uji Kling 3.0 sebelum dikunci ke `03-rab-produksi-ai.md`. Dengan reference-locking, rate kemungkinan naik → kebutuhan "2× subscribe" bisa turun ke 1×.
- **Model selector di Premier: 2.5 Turbo / 2.6 / 3.0 semua bisa dipilih, kredit beda per model.** Per klip 5 dtk 1080p: 2.5 Turbo ~35 kredit (tanpa audio), 2.6 ~55 kredit (tanpa audio), 3.0 = 60 kredit (12/dtk, ADA native audio + 4K). **Audio bawaan HANYA di 3.0** → "video+audio production-ready" wajib 3.0. Plafon production-ready per 1 subscribe Premier = 666 dtk (~11 mnt) teoretis; **patokan ~7 menit @65% accept**. Kebutuhan riil ~90 dtk → satu subscribe 4–7× cukup.
- **Keputusan Kling vs Luma Labs: TETAP KLING 3.0 untuk explainer ini.** Luma (Ray3/Ray3.14 di Dream Machine 2.0) unggul di HDR/color-grading, koherensi spasial 3D, char-reference multi-shot, dan lebih murah/klip — TAPI semua itu bobot rendah–sedang untuk proyek ini. Luma kalah di DUA kebutuhan inti: (1) **tidak ada audio bawaan terintegrasi** (Kling punya; user wajib audio; pemimpin audio = Veo 3, bukan Luma), (2) **render wajah manusia lebih lemah** (Kling "industry-leading facial rendering"). Luma dicatat hanya sebagai kandidat FALLBACK (char-reference kuat) pengganti/pendamping Seedance — tidak diadopsi. Detail + sumber: memory `semesta-kling-pricing-models.md`.
